# Аналитик FMCG (ClickHouse)

Навык для работы с синтетической витриной FMCG в ClickHouse. Перед запросами **выбери источник данных `fmcg`** в левой панели (или убедись, что активна эта БД).

## Схема

| Таблица | Назначение |
|---------|------------|
| `fact_sales` | Факты продаж: дата, магазин, товар, количество, выручка, скидка, промо |
| `dim_product` | SKU, бренд, категория, СТМ |
| `dim_category` | Категории и отделы |
| `dim_brand` | Бренды и производители |
| `dim_store` | Магазины: регион, город, канал, формат |
| `dim_promo` | Промоакции и механики |

У таблиц и колонок в ClickHouse есть **русские COMMENT** — читай их через `system.tables` / `system.columns` или метаданные DB-GPT при построении SQL.

## Рабочий процесс

1. **Подключение** — база `fmcg` (ClickHouse). Если не выбрана: попроси пользователя выбрать источник «fmcg».
2. **Уточни задачу** — период, регион, категория, KPI (выручка, штуки, маржа, доля промо).
3. **SQL** — пиши эффективный ClickHouse SQL (фильтры по дате, `PREWHERE`, агрегаты, `JOIN` к справочникам по ключам).
4. **Ответ** — таблицы, краткие выводы и рекомендации **на русском**.

## Типовые запросы

- Топ категорий / брендов по выручке за период
- Динамика продаж по неделям и регионам
- Доля промо и эффект скидок
- Сравнение форматов магазинов (hypermarket vs discounter)
- Private label vs national brand

## Важно

- Язык ответа — **русский** (интерфейс и запрос пользователя).
- Не выдумывай названия таблиц: только `fmcg.*` из схемы выше.
- При больших выборках используй `LIMIT` и агрегации, не выгружай миллионы строк в чат.
